# 多文件任务：计划、分工、检查点与最终总结

这一章讲怎么让 Claude Code 处理涉及多个文件的任务：通过拆阶段、设检查点、明确边界来控制风险，避免一口气改完导致 diff 无法审查。

## 使用场景

实现一个小功能要同时动前端、后端、测试和文档；改一条数据结构牵扯多个模块；修一个跨层 bug；或者让多个 agent 分头处理同一仓库里的不同文件——这些都是典型的多文件任务。关键是先拆阶段、明确每个阶段的文件边界，每个阶段做完就验收一次。

⚠️ 多文件任务更容易误改别人负责的东西。必须明确哪些路径允许修改、哪些不能碰。多个 agent 同时工作时，尤其不能随意回退、删除或覆盖不属于自己的改动。

普通终端命令：

```bash
cd /path/to/project
claude --permission-mode plan
```

Claude Code 对话输入：

```markdown
这是一个多文件任务。请先计划，不要修改。
目标：[写明目标]
允许修改：[列出文件或目录]
禁止修改：[列出文件或目录]
协作约束：还有其他 agent 在工作，不要回退、删除或覆盖无关文件。
请输出阶段计划、每阶段涉及文件、检查点和验收标准。
```

## 推荐提示词模板

Claude Code 对话输入：

```markdown
请按多文件任务流程工作。

任务目标：
[功能或修复目标]

允许修改范围：
- [文件 A]
- [目录 B]

禁止修改范围：
- [文件或目录]
- 其他 agent 负责的文件

阶段要求：
1. 先只读分析，列出影响面
2. 给出分阶段计划
3. 每阶段开始前说明将修改哪些文件
4. 每阶段结束后给出检查点总结
5. 运行该阶段最小验证
6. 最后给出最终总结：改了什么、没改什么、验证结果、剩余风险

高风险动作必须先问我：
删除文件、批量重构、改权限、安装依赖、提交代码、联网、处理密钥。
```

阶段确认：

Claude Code 对话输入：

```markdown
同意执行第 1 阶段。请只修改第 1 阶段列出的文件。完成后停止并给我检查点总结，不要继续第 2 阶段。
```

## 错误示范

Claude Code 对话输入：

```markdown
把这个需求完整做完，相关地方你都可以改。
```

**问题在哪：** 没有范围、没有阶段、没有检查点，最后改出一堆你审不过来的东西。

Claude Code 对话输入：

```markdown
如果发现别的 agent 改得不对，你直接覆盖掉。
```

**问题在哪：** 多 agent 协作中擅自覆盖别人的改动会导致工作丢失。应该先报告冲突，等人来决策。

普通终端命令：

```bash
claude -p "实现整个支付改版，自动安装依赖并提交"
```

**问题在哪：** 支付改版本身就是高风险的多文件任务，装依赖和提交代码还是无人值守执行。`-p` 跑完就退出，没有审查环节。

## 正确示范

Claude Code 对话输入：

```markdown
目标：给用户列表增加"禁用用户"按钮。
允许修改：
- src/users/UserList.tsx
- src/users/api.ts
- tests/users/UserList.test.tsx
禁止修改：
- src/auth
- 数据库迁移文件
- 其他 agent 正在编辑的文档目录

请先只读分析，给出阶段计划。不要修改代码。
```

Claude Code 对话输入：

```markdown
同意第 1 阶段：只改 API 调用封装和对应测试。完成后运行 users 相关测试并停止。
```

普通终端命令：

```bash
npm test -- tests/users/UserList.test.tsx
```

最终总结可以要求固定格式：

Claude Code 对话输入：

```markdown
请给最终总结，格式为：
1. 修改文件
2. 每个文件改了什么
3. 验证命令和结果
4. 没有修改的范围
5. 剩余风险和建议下一步
```

## 练习任务

1. 设计一个涉及 3 个文件的小需求。
2. 写出允许修改和禁止修改的路径。
3. 让 Claude Code 先给阶段计划。
4. 只批准第一阶段，要求完成后停止。
5. 审查检查点总结，再批准下一阶段。
6. 最后要求 Claude Code 用固定格式总结修改和验证结果。

进阶练习：模拟多 agent 协作，指定某些文件"由其他 agent 负责"，检查 Claude Code 是否避开了这些文件。

## 验收标准

- 多文件任务开始前有影响面分析和分阶段计划。
- 每阶段都列出了将修改的文件，并且得到了确认。
- 实际修改没有超出允许范围，没有碰其他 agent 负责的文件。
- 每阶段都有检查点总结和最小验证。
- 最终总结包含修改文件、修改内容、验证结果、未修改范围和剩余风险。
- 高风险操作均未擅自执行。
