# 必备编程基础：终端、路径、项目结构

Claude Code 好不好用，很大程度取决于你是不是在正确的目录下启动，以及你能不能跟它说清楚项目长什么样。终端操作、路径概念和项目根目录的判断，是后续所有进阶用法的地基。

⚠️ 不确定自己在哪个目录时，别让 Claude Code 删文件、大范围改代码、装依赖或提交。路径搞错了，影响的可能是完全不相干的项目。

## 使用场景

你需要掌握这些基础，才好完成下面的事：

- 在正确的项目根目录启动 Claude Code。
- 让它找到入口文件、依赖文件和测试目录。
- 判断某条命令该在项目根目录跑、子目录跑还是随便在哪跑。
- 明确告诉 Claude Code 哪些目录可以改，哪些只能看。

**常见项目结构概念：**

- **项目根目录：** 通常包含 `.git`、`README`、依赖文件和配置文件的那个目录。
- **入口文件：** 程序启动时最先执行的文件，或者框架约定加载的文件。
- **依赖文件：** 记录第三方库和版本号的文件，比如 `package.json`、`requirements.txt`、`pyproject.toml`、`Cargo.toml`。
- **测试目录：** 常见名字有 `test`、`tests`、`__tests__`、`spec`。
- **配置文件：** 比如 `.env.example`、`tsconfig.json`、`vite.config.js`、`pytest.ini`。

普通终端命令：

```bash
pwd
ls
find . -maxdepth 2 -type f
```

Windows PowerShell 普通终端命令：

```powershell
Get-Location
Get-ChildItem
Get-ChildItem -Recurse -File -Depth 2
```

## 推荐提示词模板

Claude Code 对话输入：

```markdown
请只读分析当前项目结构，不要修改文件。

请找出：
- 项目根目录判断依据
- 入口文件候选
- 依赖文件
- 测试目录
- 配置文件
- 你还不能确定的信息

输出格式：
- 用表格列出路径、类型、判断依据。
- 最后给出我下一步应该阅读的 5 个文件。
```

Claude Code 对话输入：

```markdown
请帮我确认当前目录是否适合执行任务。

任务：修改登录页文案。
范围：只允许修改前端页面和相关测试。
禁止：删除文件、批量重构、安装依赖、联网、提交代码、处理真实密钥。

请先说明你认为相关文件在哪里，再等我确认。
```

## 错误示范

普通终端命令：

```bash
claude
```

**问题在哪：** 命令本身没错，但如果你不知道当前在哪个目录，可能直接在错误的项目里启动了。启动前先跑一下 `pwd` 和 `git status`。

Claude Code 对话输入：

```markdown
把 src 下面都整理一下。
```

**问题在哪：**

- `src` 可能非常大。
- "整理"没有具体目标——是改命名、调结构还是删文件？
- 没说能不能重命名、移动或删除文件。

## 正确示范

普通终端命令：

```bash
pwd
git status
ls
```

Claude Code 对话输入：

```markdown
请先确认当前目录是否为项目根目录。不要修改文件。

请检查：
1. 是否存在 Git 仓库。
2. 是否存在 README 或依赖文件。
3. 是否存在源代码目录和测试目录。
4. 如果当前不是根目录，请说明我应该切换到哪个目录。
```

Claude Code 对话输入：

```markdown
请只修改 `docs/usage.md` 中的运行说明，不要改代码。

限制：
- 不删除文件。
- 不移动文件。
- 不改权限。
- 不安装依赖。
- 不提交代码。

完成后请告诉我修改了哪些段落，以及我应该如何检查。
```

## 路径基础

绝对路径从系统根位置开始，比如 `/home/user/project` 或 `C:\Users\user\project`。

相对路径从当前目录算起，比如 `src/main.js`、`../README.md`。

当前目录很重要——同一个相对路径，在不同目录下指向的是不同文件。

普通终端命令：

```bash
pwd
cd ..
pwd
cd -
```

Windows PowerShell 普通终端命令：

```powershell
Get-Location
Set-Location ..
Get-Location
```

Claude Code 对话输入：

```markdown
请解释 `src/app`、`./src/app`、`../src/app` 在当前目录下分别指向哪里。不要修改文件。
```

## 练习任务

1. 找一个练习项目。
2. 分别在项目外层、项目根目录、项目子目录运行 `pwd` 或 `Get-Location`。
3. 判断哪个才是该启动 Claude Code 的目录。
4. 让 Claude Code 只读列出项目结构。
5. 自己手动记下入口文件、依赖文件、测试目录和配置文件。

普通终端命令：

```bash
pwd
git status
find . -maxdepth 2 -type f
```

Claude Code 对话输入：

```markdown
请根据当前目录生成项目结构速查表。只读分析，不要修改文件。表格列为：路径、作用、我学习时应该关注什么。
```

## 验收标准

学完本章，你应该能做到：

- 解释项目根目录、入口文件、依赖文件、测试目录分别是什么。
- 用普通终端命令确认当前路径。
- 判断 Claude Code 是不是在正确目录里跑。
- 把修改范围限定到具体目录或文件。
- 意识到路径不清时执行修改操作的风险。
