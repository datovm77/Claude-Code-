# Claude Code 进阶学习路线

这套教材假定你已经装好了 Claude Code。我们不再讲安装，而是把 Claude Code 当作一个能读项目、改文件、跑命令、配合 Git 干活的编程助理，帮你系统地用起来。

建议按顺序学：

1. `00`：建立学习路线和安全习惯。
2. `01`：搞清楚 Claude Code 怎么工作——工具、权限、环境自检。
3. `02`：补齐终端、路径、项目根目录这些基础。
4. `03`：掌握 Git、diff、commit 和回退。
5. `04`：学会写高质量提示词，把模糊想法变成可执行任务。

⚠️ **高风险操作提醒**：删文件、大范围重构、改权限、装依赖、提交代码、联网、碰密钥——这些操作在做之前，都要先看计划和 diff，确认影响再放行。后续每章都会在开头标注该章特有的风险点。

## 使用场景

**适合你的情况：**

- 能打开终端跑 `claude`，但不知道怎么系统练习。
- 想拿 Claude Code 处理真实项目，不只是问概念题。
- 经常碰到"它改了一堆文件但我看不懂改了什么"的局面。
- 想建立从提问、执行、检查、提交到复盘的完整流程。

**不建议这么干：**

- 直接把生产仓库扔给 Claude Code 大面积修改。
- 不了解项目根目录和 Git 状态就要求批量重构。
- 把 API Key、SSH 私钥、数据库密码直接贴进对话。

普通终端命令：

```bash
pwd
git status
claude
```

Claude Code 对话输入：

```markdown
请先阅读当前项目结构，不要修改文件。请告诉我这个项目的入口文件、依赖文件、测试目录和你建议我先学习的 3 个文件。
```

## 学习路径

**第一阶段：会定位。** 你要能回答"我在哪个目录""项目根目录在哪""Claude Code 能看到哪些文件"。

**第二阶段：会约束。** 你要能把任务写清楚——背景、目标、范围、限制、验收标准、输出格式，一个都不少。

**第三阶段：会检查。** 你要能看懂 `git diff`，知道 Claude Code 改了什么、为什么改、有没有越界。

**第四阶段：会交付。** 你要能让 Claude Code 跑测试、整理变更摘要，确认没问题后再提交。

**第五阶段：会复盘。** 你要能把好用的提示词、踩过的坑和项目约定沉淀成笔记或 `CLAUDE.md`。

## 推荐提示词模板

Claude Code 对话输入：

```markdown
你是我的 Claude Code 实战教练。请基于当前目录帮我制定一个 30 分钟练习计划。

背景：
- 我已经安装 Claude Code。
- 我想学习进阶实战，而不是安装教程。

目标：
- 理解当前项目结构。
- 练习只读分析、局部修改、查看 diff、运行检查。

范围：
- 只分析当前项目。
- 不删除文件，不批量重构，不安装依赖，不提交代码。

输出格式：
- 先列学习步骤。
- 再列每一步要输入的命令或提示词。
- 最后列验收标准。
```

Claude Code 对话输入：

```markdown
请为我执行一个安全的学习任务：只读取当前项目，不修改文件。请找出项目根目录、主要语言、依赖文件、测试命令候选项，并说明你依据了哪些文件。
```

## 错误示范

Claude Code 对话输入：

```markdown
帮我把这个项目优化一下，顺便提交。
```

**问题在哪：**

- 没说优化什么。
- 没限定改动范围。
- "顺便提交"——提交代码前必须先看 diff 和测试结果，不能顺手就来。
- 没说能不能装依赖、联网、删文件。

Claude Code 对话输入：

```markdown
这个项目有点乱，你直接重构所有文件。
```

**问题在哪：**

- 全仓库重构风险极高，改完你根本审不过来。
- 没有验收标准，改完怎么算"不乱"？

## 正确示范

Claude Code 对话输入：

```markdown
请先只读分析当前项目，不要修改文件。

请完成：
1. 判断项目根目录是否正确。
2. 找出入口文件、依赖文件、测试目录。
3. 找出最适合新手阅读的 5 个文件。
4. 列出你建议的第一个小修改任务。

限制：
- 不删除文件。
- 不安装依赖。
- 不联网。
- 不提交代码。

输出：
- 用表格列出文件路径和判断依据。
- 最后给出下一步建议。
```

Claude Code 对话输入：

```markdown
请做一个最小范围的修改：只更新 README 中的运行说明，不改代码文件。

执行前请先给计划；执行后请给：
- 修改过的文件列表
- 关键 diff 摘要
- 是否需要我运行命令验证
```

## 练习任务

1. 在一个非生产项目中启动 Claude Code。
2. 让 Claude Code 只读分析项目结构。
3. 根据分析结果，挑一个只改文档或只改测试的小任务。
4. 要求 Claude Code 先给计划，再动手改。
5. 改完后用普通终端命令看 diff。

普通终端命令：

```bash
git status
git diff
```

Claude Code 对话输入：

```markdown
请根据刚才的修改生成一份变更摘要，但不要提交代码。摘要需要包含：改了什么、为什么改、如何验证、还有哪些风险。
```

## 验收标准

学完本章，你应该能做到：

- 说清这套教材每章解决什么问题。
- 解释为什么要先分析项目结构，再让 Claude Code 动手改。
- 在高风险操作前主动要求 Claude Code 停下来等确认。
- 识别常见的高风险场景：删文件、大范围重构、改权限、装依赖、提交代码、联网、碰密钥。
- 写出一个包含背景、目标、范围、限制、验收标准的任务提示词。
